# cache 的工作原理

为了解决 CPU 和主存之间速度不匹配的问题，从而引入 Cache，将某些主存块复制到 Cache 中。

# 主存地址结构划分

主存块 的内容非常 “纯粹”，它只包含实际的数据或指令代码：
一个主存块就是一个固定大小的、连续的物理存储单元，里面存放着原始的二进制信息。
主存块的地址是由硬件机制决定和解释的。（主存块地址在复制主存块内容时，由硬件获取）
主存地址： 主存块号 + 块内地址

cache 复制主存内容时，主存地址被硬件拆分出 tag，至于组号、行号都不放在 cache 行里。

# 1. 全相联映射

cache 的所有行均可用于存放主存任何一块数据
缺点：查找最慢，当 cache 块装满的时候，可能需要遍历所有的 cache 行
主存地址结构划分：
主存块号 块内地址
22 位 6 位
查找策略：
假如访问地址为：1…1101 001110

用前 22 位依次与 cache 块标记进行对比
如果标记匹配且有效位为 1，则 cache 命中，访问块内地址为 001110 的单元
若 cache 不命中或有效位为 0，则正常访问内存

# 2. 直接映射

每个主存块只能放到某个固定的 cache 行，每个主存块所属的 cahe 行是 主存块号 % cache 总行数
缺点：所以多个块会映射到同一行，这样会产生不必要的换入换出，因为即使 cache 有空行，也不能利用。
主存地址结构划分：

主存块号 块内地址
主存字块标记（19 位） + cache 行号（3 位） = 22 位 6 位
查找策略：
假如访问地址为：0…01000 001110
根据主存块号后 3 位确定 cache 行号
如果主存块号前 19 位与 cache 标记匹配且有效位为 1，则 cache 命中，访问块内地址为 001110 的单元
若 cache 不命中或有效位为 0，则正常访问内存

# 3. 组相联映射

首先要对所有的 cache 行进行分组，每个主存块只能放到固定的 cache 组，每个主存块所属的 cahe 组是 主存块号 % cache 总组数

主存地址结构划分：
主存块号 块内地址
主存字块标记（20 位） + cache 组号（2 位） = 22 位 6 位

假如访问地址为：1…0101 001110

查找策略：
根据主存块号后 2 位确定 cache 组号
如果主存块号前 20 位与 cache 分组内某行标记匹配且有效位为 1，则 cache 命中，访问块内地址为 001110 的单元
若 cache 不命中或有效位为 0，则正常访问内存

# cache 容量计算


Cache 每一行的内容： (计数位 + 脏位) + 有效位 + 标记 + 存储的内容
即每个 cache 行都会对应一个标记项，用于标记当前 cache 行保存的数据状态 即 控制算法位（(计数位 + 脏位) + 有效位 + 标记）+ 数据位（存储的内容）
【计数位】是用于 LRU 算法的 也叫替换控制位 比如是 2^n 路组相联，就需要 n bit 的计数位
【脏位】是用于 Cache 写策略的 只有当采用写回法时，该位生效并占一位
【有效位】为 1 说明该 Cache 行的内容有效，为 0 则不然
【标记】Tag 是用于与地址进行比对的关键字 主存字块标记位数
【存储的内容】就是对应的主存页面的副本 即 cache 块
由于 Cache 是相联存储器，是按内容寻址的，并没有划分地址结构
cache 的容量 = cache 行数 * cache 位数 = cache 行数 * （cache 控制算法位 + cache 数据位）

# 题目详解

  1. 假设主存地址位数为 32 位，按字节编址，主存和 Cche 之间采用全相联映射方式，主存块大小为 1 个字，每字 32 位，采用回写法 (Vrite Back) 方式和随机替换策略，则能存放 32K 字数据的 Cache 的总容量至少应有多少位 。
    • 1536K
    • 1568K
    • 2016K
    • 2048K

    字节编址 每字 32 位 即 4B 则块内地址为 2 位，数据位为 32 位
    全相联映射方式 存放 32K 字数据则 主存地址 划分为 tag 位 + 块内地址 = 32 位 ，则 tag 位 = 30 cache 行数 = 32K
    采用回写法 (Write Back) 方式和随机替换策略 则 有效位 =1 ， 计数位 = 0 ， 脏位 =1
    cache 行位数 = (计数位 + 脏位) + 有效位 + 标记 + 存储的内容 = 64
    则 cache 总容量 = cache 行数 * cache 行位数 =32K * 64bit =2048K bit.
